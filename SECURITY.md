# Security

The K2s project treats security vulnerabilities seriously and aims to respond quickly when action is required.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

If you believe you have found a security issue or vulnerability in K2s, report it to:

```text
dieter.krotz@siemens-healthineers.com
```

Please include as much information as possible to help with analysis:

- Description of the vulnerability.
- Affected component, version, commit, or branch.
- Affected code path or file, if known.
- Steps to reproduce.
- Impact assessment.
- Exploit code or proof of concept, if available.
- Suggested mitigation or fix, if available.

## Response Process

The maintainers will review the report, determine impact, and coordinate a fix or mitigation. Security-sensitive details may be handled privately until a fix is available.

## Security Team

The security team currently consists of the K2s maintainers listed in [MAINTAINERS.md](MAINTAINERS.md).

Additional security information is also available in the project documentation: [Security](docs/index.md#security).
